# 메모리 주소
- 폰노이만 아키텍처 : RAM에 있는 [코드(텍스트) + 데이터 형식]
- 주소는 어딘가에 들어 있는 개념이 아니라 **체계** 의 개념임

# 변수
## 전역변수
- 컴파일 될 때 초기화
- Life Cycle : 프로그램이 시작될 때 ~ 끝날 때까지 생존
- Scope : 해당 프로그램 외부까지 존재
- Unique(유일함)
- RAM 안의 Data
## 지역변수
- Life Cycle : 해당 블럭이 시작 될 때 ~ 끝날 때까지 생존
- 해당 블럭 안에서만(Scope)
- RAM안의 Stack
## 동적할당
- ex) malloc, new
- 런타임(프로그램이 실행 되는 시기)에 생성되는 변수
- RAM안의 Heap

# 파일
- 영속성(Persistence)을 해결하기 위해(컴퓨터를 껐다 켰을 때 RAM은 초기화)
- stack, heap에 저장된 값은 저장하지 않지만, 전역변수는 초기값을 저장해놓음
- 초기화가 안 된 전역변수는 0으로 초기화해서 저장

# Loading / Relocation
- Loading(탑재) : 실행파일을 메모리에 탑재하는 과정
- Relocation(재배) : 실행전에 주소값을 다 바꿈
- Virtualization(가상화) : 실행 할 때 그때그때 마다 주소값을 바꿈(동적인 방법)

# Booting
- 컴퓨터 전원을 켰을 때 커널이 메모리에 탑재되는 과정

# 검색
- <Key, Value> 형식 : key를 검색하면 value를 리턴하는 방식
- key를 sort 할 수 있으면 이진검색 가능
- Hash 방식 : key값이 들어왔을 때 hash 과정을 통해서 한번에 (시간복잡도 O(1))
  찾을 수 있는 방법 -> 배열 이용(**연속된** 공간이 필요함)
