# Use-Cases
- use case : 요구사항 식별/작성
- 시스템을 사용하는 상황의 스토리
- 요구사항을 이해하고 표현하는 좋은 방법
## Use Case Model
- 모든 use cases, 시스템의 기능과 환경에 대한 모델
- 시스템 이용자가 시스템을 어떻게 이용한지 기술한 것
- use case는 text document다 (diagram이 아님)
## Actor
- 행동을 하는 것(사람, 시스템, 단체 등)
- 시나리오에서의 주어/목적어
- primary actor : 시스템 서비스 사용으로 달성되는 목적이 있는 사용자
- supporting actor : SuD에 서비스를 제공(manager)
- offstage actor : 시스템이 운영되도록 하는 주체(admin)
- use case와 use case diagram은 다르다
