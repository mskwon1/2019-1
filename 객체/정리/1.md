# Introduction

## Analysis and Design

### Analysis

- 문제와 요구사항을 찾아내는 것(해결법에 집중 X)
- Requirements Analysis, Object Analysis
- Object Oriented Analysis : 개념, 객체를 찾고 설명하는 것에 중점을 둠

### Design

- 요구사항을 충족하는 개념적 해결방법(구현 X)
- Object Oriented Design : 소프트웨어 객체와 그들이 어떻게 상호작용하는지에 중점

## 절차

### Define Use Cases

- 관련된 도메인 프로세스들의 설명

### Define Domain Model

- 개념, 속성, 연관성 등의 식별
- 도메인 컨셉/객체 SHOW

### Define Interaction Diagrams

- 소프트웨어 객체간의 메세지 흐름 표시, 메소드 발동 시기 표시

### Define Design Class Diagrams

- 클래스 정의에 대한 정적인 뷰를 만듬
- 각 클래스의 속성, 메소드 표시
