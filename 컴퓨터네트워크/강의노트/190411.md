# Media Access Control(MAC)
## Random Access
- station별로 전송에 정해진 시간이 없음
- 어떤 station이 send를 할지에 대한 규칙이 없음 -> station들이 medium에 접근하기 위해 경쟁
- send할지 안할지를 프로토콜에 의해 정해진 procedure를 통해 결정 -> medium이 idle/busy인지에 따라 결정
- 충돌 : 2개 이상의 station이 보내기를 시도하면, 접근 충돌이 생겨 프레임이 파괴되거나 수정됨
### ALOHA
- 가장 오래된 랜덤접근방식(하와이 대학교에서 개발)
- radio LAN으로 설계되었으나, 아무 medium에 사용 가능
- 잠재적 충돌가능성이 존재
  : 한 station이 전송을 시도 할 때, 다른 station도 전송시도를 할 수 있음 -> 충돌발생
#### Pure ALOHA
- 충돌이 발생해 2개 이상의 station이 타이머가 지난 후 다시 전송을 시도 했을 경우, 또 충돌 발생
- Random Backoff로 해결 : timeout 발생시 각 station별로 랜덤 시간만큼 기다렸다가 재전송 시도함
  -> T_B = random backoff time
- K_max번만큼 재전송을 시도 한 이후에는, 포기하고 나중에 하도록 해야 함
- T_P : sender -> receiver의 최대 Propagation delay
  -> Time-out period = 2*T_P
- R : 0 < R < 2^k -1
  k = 실패한 전송의 숫자
- T_fr = frame을 전송 할 때 걸리는 평균 시간
  backoff time = T_B = R*T_P, T_B = R*T_fr
- Vulnerable Time : 충돌의 가능성이 있는 시간 = 2 * T_fr
- Throughput(처리량) : S = G * e^-2G
  S = 성공적으로 전송된 프레임의 평균 숫자
  G = one frame transmission time동안 생산된 평균 프레임 수
  e = 2.71
  S_max = 0.184 (G = 1/2일때)
#### Slotted ALOHA
- Pure ALOHA는 station이 언제 보낼 수 있는지에 대한 규칙이 정의가 안돼있으므로 Vulnerable time = 2*T_fr
- Slotted ALOHA에서는 시간을 T_fr의 타임슬롯으로 나누고, 각 노드들을 time slot의 시작 지점에서만 보내도록 함
- Node가 보내는데에 실패하면, 다음 time slot의 시작까지 기다려야 함
- 2개이상의 Node가 같은 time slot에서 동시에 보내기를 시도하면 충돌 발생
- Vulnerable Time = T_fr
- Throughput : S = G * e^-g
  S_max = 0.368 (G = 1일때)
