# R의 특징
- 무료로 사용 할 수 있는 오픈소스, 성능이 좋음
- OS 관계 X
- Interpreter 언어(Compile 과정이 필요 없음), 실행 속도는 그만큼 느려짐(체감되는 정도는 아님)
- 그래픽 처리 능력이 좋음(그래프가 예쁘게 그려짐)
- 외부의 어떤 데이터도 쉽게 읽어 들일 수 있음

# R 실행
- etc 폴더 RConsole(R 콘솔에 대한 개인 환경 설정) , Rprofile.site(R을 시작 할 때 실행 될 것 지정)
- setwd("디렉토리"), getwd() -> 워킹디렉토리 설정
- .libPaths() : 라이브러리 패쓰 확인 ... Program Files에는 표준 라이브러리, 내 문서에는 내가 설치 한 라이브러리
  인자로 디렉터리를 넣어서 path 지정 가능
- .First() : 세션이 시작 할 때마다 우선적으로 실행 - Rprofile.site를
- .Last() : 세션이 끝날 때 마다 실행
- 스크립트 파일 : 배치 파일의 개념
- q() : R을 끝낼 때 사용하는 명령어 (닫기버튼과 같음)

# Vector
- 데이터의 집합(벡터 형태)
- c() 함수로 간단하게 벡터 생성 가능

# 작업공간/기본함수
- .RData 파일 : 현재 작업중인 새엉된 변수와 함수를 저장함, 다음에 R을 실행할때 자동으로 load됨(그래프는 저장 X -> 함수 이용)
- 작업 중간에 이미지 저장 -> **save.image()** : .RData 파일에 현재 상태 저장
- ls() : 현재 생성된 변수/함수들의 리스트 반환 ... 아무것도 없을때 character(0) 반환
  ls(pattern="문자열") : 해당 문자열이 들어간 변수/함수의 리스트를 반환함
- rm() : 해당 변수/함수를 작업공간에서 삭제(되돌릴 수 없음), 인자로 list = ls() 할 시 전체 삭제
- history() : 최근 25개까지의 명령어 라인을 보여줌
  history(n) : n개의 명령어, history(Inf) 모든 명령어 라인 보여줌
- .Last.value : 바로 직전에 실행했던 명령어의 실행 결과 반환 ... 오래 걸리는 함수를 실행한 결과를 얻을 때 등 유용
- save() : 특정 변수 저장, save(변수, file="파일명") 형식으로 저장 가능(rda 타입)
- load("파일명") 시 해당 파일의 변수/함수 불러옴
- search() : 현재 어떤 패키지들이 사용 가능한지 확인 ... 같은 이름의 변수/함수가 중복되는 경우 맨 앞의 패키지 실행
- 패키지를 사용하려면 해당 패키지가 적재(load)되어 있어야 함 -> 표준 패키지에 있는 내용도 사용하기위해서는 적재 필요
- library("패키지명") : 해당 패키지를 현재 메모리에 적재
- detach(package:패키지명) : 해당 패키지를 메모리에서 제거
- install.packages("패키지명") : CRAN에 있는 패키지를 설치, 적재는 따로 해줘야함
- help(package = "패키지명") : 해당 패키지의 도움말 불러오기, library(help=패키지명)도 같은 기능
  -> help(함수명) = ?함수명과같은 역할, ??함수명 사용시 해당 함수명과 유사한 함수를 찾아서 help값 리턴
- example(함수명) : 해당 함수의 사용법 출력
- args(함수명) : 해당 함수의 인수에 대해서 출력
- apropos() : 생성했던 변수 이름이나 사용하고자 하는 함수 이름의 일부만을 이용해 검색
  -> 단순 문자열과 정규표현식 모두 사용가능
- data(package="패키지명") : 해당 패키지에 있는 데이터셋 정보 출력
  data(변수, package="패키지명") : 해당 데이터셋을 변수에 저장
  head(), tail()으로 앞부분 n개, 뒷부분 n개만 출력가능
