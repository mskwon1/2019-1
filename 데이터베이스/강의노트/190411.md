# 관계 데이터 구조
## 애트리뷰트와 도메인
- 테이블과 릴레이션은 비슷한 개념
- 도메인 : 애트리뷰트가 취할 수 있는 값들의 집합
  -> 단순 도메인 : 단순 애트리뷰트 - 원자값
  -> 복합 도메인 : 복합 애트리뷰트 - 복합값 ex) 날짜:<연,월,일>
- 애트리뷰트 : 도메인의 역할 이름, 이름이 모두 달라야 함
- 애트리뷰트 이름과 도메인 이름이 같을 수 있음
## 릴레이션
- 수학적 정의 : 릴레이션 R = 도메인의 카티션 프로덕트의 부분집합
  -> n-tuple <v1,v2,...,vn>의 집합 R = {<v1,v2,...,vn> | vi in Di, i = 1,2,...,i}
  -> n = R의 차수, 튜플의 수 = 카디널리티
- 개념적 정의 : 릴레이션 스킴 + 릴레이션 인스턴스
### 릴레이션 스킴
- 릴레이션 내포(Intension)
- 릴레이션 스키마 : 릴레이션 이름 + 애트리뷰트 이름
- 정적 성질 : 시간에 무관
### 릴레이션 인스턴스
- 릴레이션 외연(Extension)
- 어느 한 시점에 릴레이션 R이 포함하고 있는 튜플들의 집합
- 동적 성질 : 갱신 연산에 의해 시간에 따라 변함
- 튜플 = 순서쌍
### 특성
- 튜플의 유일성 : 릴레이션 = 서로 다른 튜플의 집합
- 튜플들의 무순서 : 릴레이션 = **추상적** 개념, 테이블 = 튜플의 리스트, 릴레이션의 Snapshot, **구체적** 개념
- 애트리뷰트들의 무순서 : 릴레이션 스킴 = 애트리뷰트들의 집합, 튜플 = <attr:value>쌍의 집합
- 애트리뷰트는 원자값 : 분해 불가능, **정규화 릴레이션(비정규화 릴레이션은 분해를 통해 정규화 - 저장되는 값이 원자화 되도록)**,
                        널값은 원자값으로 처리, 단순 도메인, 복합 도메인(값을 하나의 단위로 취급)
## 관계 데이터베이스
- 데이터베이스를 시간에 따라 그 내용이 변할 수 있는 테이블 형태로 표현
- 관계 데이터베이스 스키마 = 릴레이션 스키마

# 관계 데이터 제약(Key에 대한 제약)
## 기본키
- 키 : 튜플을 유일하게 식별 할 수 있는 애트리뷰트 집합
- 후보키 : 릴레이션 R에 대한 애트리뷰트들의 집합,
            해당 애트리뷰트/애트리뷰트의 집합이 유일성, 최소성(최소의 애트리뷰트들로 구성)을 만족
- 슈퍼키 : 유일성은 만족하지만 최소성을 만족하지 않는 키의 집합
