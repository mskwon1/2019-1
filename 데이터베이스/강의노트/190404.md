# ERD의 용도
- conceptual design용으로 개발, logical design의 용도로도 사용가능
- Conceptual : PK는 표현, FK는 직접 표현 X
  -> 단일선-이중선 표기법, Min-Max Notation, Crow's foot notation, Barker's notation
- Logical Design : PK와 FK 모두 표현
  -> IDEF1X, EER diagram(mySQL)

# ERD의 다른 표기법
- 단일선-이중선 표기법 : Relationship 기준
- Min-Max 표기법 : Entity Type 기준 - (0,1) 부분참여 (1,1) 전체참여 (0,n) 부분참여 (1,n) 전체참여 -> min값으로 구분
- 변환(Min-Max -> 단일선-이중선) : min은 남기고 max를 반대방향으로 이동
## Crow's Foot Notation : 가장 많이 사용함
- Crow's foot = many, straight line = one, empty circle = optional
- EER에서는 crow's foot notation을 logical design으로 사용(n-m 관계 없고, 외래키는 따로 설명), Identifying Relationship : 실선, non : 점선
## Barker's notation : 오라클에서 사용
- Unique IDentifier (#)
- Mandatory (\*)
- Optional (O)
- 전체 참여 = 실선, 부분 참여 = 점선
- UID Bard |< 형태... 해당 엔티티의 필드가 다른 끝의 엔티티 키의 component로 쓰였다는 뜻
- 데이터 중복을 어떠한 경우에도 허용하지 않음 -> 외래키의 개념으로 관계를 사용(타겟 리스트에는 attr로 안들어감)
- binary relationship만 형태
