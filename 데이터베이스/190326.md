# 관계 데이터 모델
- 개념적 데이터 모델 : 관계 데이터 모델(= 테이블)
- 논리적 데이터 모델 : Entity Relationship data model(현실 세계를 추상적으로 표현)
- 관계 데이터 모델, 계층 데이터 모델, 네트워크 데이터 모델 중 **관계 데이터 모델이 제일 많이 쓰임**
- DB : 릴레이션(테이블)의 집합
- 관계 스킴 : 개체와 관계성의 테이블 정의 , 별표*는 외래키(FK)를 의미
## ER 다이아그램에서 관계스킴으로의 변환
- 검색 효율성을 고려 : 릴레이션의 수가 적을수록 연산이 효율적 -> 릴레이션을 최대한 줄이는 방향으로
- Entity Type -> Relation으로
- 1-1 / 1-n Relationship Type -> FK(외래키)로
- n-m relationship type -> Relation으로
- 다중값 애트리뷰트 -> Relation으로
- 엔티티 타입의 유도 애트리뷰트 -> 레코드 타입에서 제외 : ER다이어그램에서만 논하는 개념, SQL에서 처리함
